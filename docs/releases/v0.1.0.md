# v0.1.0 - Local Research Workflow Workbench

This first project release packages the current local-first research workflow into a documented, test-covered workbench.

## Highlights

- Local browser UI for common project, literature, note, simulation, figure, and manuscript actions.
- CLI for workspace initialization, literature library work, reading notes, simulation checks, figure generation, and manuscript QA.
- Demo workspace at `examples/demo-project`.
- Real browser screenshots captured from the demo workspace.
- Crossref open metadata discovery and direct/open PDF URL download helpers.
- Deeper DOCX package checks for styles, page margins, and Word citation/bibliography field signals.
- External solver command logging through `simulation run-command`.
- Beginner guides in Chinese and English, plus a dedicated Chinese web UI guide.
- GitHub project metadata: license, contribution guide, security policy, changelog, roadmap, citation metadata, issue templates, PR template, and CI.

## Core Workflow

- Create a standard research workspace.
- Add or import literature metadata, discover Crossref open metadata, and download direct/open PDF links.
- Generate search-log and paper-summary notes.
- Run installed solver CLI commands with logs, then inspect, validate, summarize, and range-check simulation CSV/JSON exports.
- Generate trend, bar, error-bar, heatmap, and contour SVG figure bundles.
- Check Markdown, plain text, and DOCX manuscript drafts for structure, citation coverage, captions, references, and basic Word package signals.
- Generate writing packs, literature tables, writing dashboards, literature maps, and literature trackers.

## Verification

The release baseline has 161 unittest cases covering the main workflow modules and web action handlers.

```powershell
python -m unittest discover -v
```

## Known Limits

- Literature discovery uses open Crossref metadata and user-provided direct/open PDF URLs; it does not bypass publisher permissions, institutional login, or paywalls.
- DOCX checking inspects text and package-level signals, but it does not render Word pages or validate full institution-specific formatting.
- Simulation automation can launch installed solver CLI commands and capture logs, but it does not control commercial solver GUIs, licenses, or model setup.
- Contour plots require complete rectangular x/y/value grids.
