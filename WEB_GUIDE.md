# 本地网页界面使用教程

本文档是本项目的网页操作主教程。适合不想手敲 Python 命令、希望用本地网页完成科研工作流的人。

`USER_GUIDE.md` 是总入口和完整流程教程；本文件只讲网页怎么打开、先点哪里、结果怎么看、出错后怎么处理。以后凡是网页界面新增按钮、表单、结果卡片或交互行为，应优先更新本文件，并在必要时同步更新 `USER_GUIDE.md` 的总入口说明。

## 1. 打开网页

直接双击工具目录里的：

```text
C:\Users\YourName\Documents\科研\start_web.bat
```

它会启动本地网页服务，并打开：

```text
http://127.0.0.1:8000
```

`start_web.bat` 会依次查找 `.venv\Scripts\python.exe`、Windows 的 `py` 启动器和 `python` 命令。若提示找不到 Python，请先安装 Python 3.10+，或在工具目录里创建 `.venv`。

网页服务只在本机运行，地址是 `127.0.0.1`。默认使用 `8000` 端口；如果端口被占用，工具会自动尝试 `8001` 到 `8009`，并打开实际可用的地址。

如果浏览器打不开，先确认 `start_web.bat` 的窗口没有被关掉；窗口关闭后网页服务也会停止。

## 2. 先填项目根目录

网页里先填写你的课题目录，例如：

```text
C:\Users\YourName\Documents\fixture-study
```

如果只是想先试用，不想马上新建自己的课题，可以填写仓库自带示例课题：

```text
C:\Users\YourName\Documents\科研\examples\demo-project
```

网页会记住上次填写的课题目录，下次打开时不用重新输入。

## 3. 推荐第一次使用顺序

网页界面和推荐使用顺序可以先看这张示意图：

![科研工作流网页使用顺序示意图](workflow/web_assets/workflow-guide.svg)

第一次打开网页时，建议按这个顺序操作：

1. 填写“项目根目录”，或点击“加载示例课题”先试跑。
2. 点击“流程状态”，先看任务向导告诉你下一步该做什么。
3. 点击“扫描可用文件”，让网页自动填入常见数据、稿件和输出路径。
4. 按任务向导跳到对应功能区，逐步处理文献、笔记、仿真、图表和稿件。
5. 每完成一轮，点击“项目体检”检查还有哪些缺口。

如果你已经填好了自己的课题目录，点击“填充常用路径”，网页会自动把仿真数据、输出图目录、稿件路径、必需章节和预期图号填成常见默认值。你可以再按自己的真实文件名修改。

页面左侧还提供“展开全部”和“收起全部”。当你要连续检查多个模块时，可以先展开全部快速浏览；只想专注当前模块时，可以收起全部再打开需要处理的区域。

## 4. 页面功能区

网页不是营销页面，而是本地工作台。你可以把它理解成几个功能区：

- 项目区：新建课题、填写项目根目录、查看项目总览、运行项目体检。
- 文献区：添加或导入文献，搜索文献，检查缺 PDF、缺笔记，查看文献洞察。
- 笔记区：生成论文摘要卡、记录检索过程。
- 仿真区：预览 CSV/JSON，汇总数值范围，校验列名、数值和单位。
- 图表区：从 CSV/JSON 生成 SVG 论文图。
- 稿件区：检查论文草稿的章节、图号、引用、参考文献，以及 DOCX 包结构、图片替代文本和修订/批注痕迹。
- 写作区：生成写作素材包、文献对比表、写作看板、文献地图和追踪清单。

## 5. 运行结果怎么看

右侧“运行结果”区域可以复制结果、下载当前结果为 `.md` 文件，也可以清空后继续操作。

点击任意操作后，结果区会进入运行中状态。运行期间按钮会变灰，不能重复点击；“重跑”也会暂时禁用，避免同一个任务被重复提交。运行结束后，如果你改了路径或参数，可以点“重跑：某个操作名”重新执行刚才那个操作。

每次操作成功后，“运行结果”标题下方会显示最近成功的操作名和时间，并保留最近几次成功操作的小标签，方便你确认刚刚完成的是哪一步。这些成功记录会保存在当前浏览器里，刷新页面后仍会恢复；点击“清空”会同时清掉这段历史。

如果某个操作没有成功，右侧结果区上方会显示“操作需要处理”提示卡。它会先告诉你最可能该检查什么，例如项目根目录、文件路径或必填字段。提示卡里的“去对应区域”按钮会展开并跳到最可能需要修改的功能区，下方仍会保留原始错误文本，方便复制给别人排查。

## 6. 必填字段和预检查

网页会在发送前先检查明显缺失的内容。比如没有填写项目根目录，或者导入 CSV、预览仿真数据、生成图、检查稿件时没有填写对应文件路径，网页会直接提示你补哪个字段，并自动打开、高亮、聚焦到对应输入框。

生成 SVG 图时，网页还会提前检查输出目录和文件名。

网页中带“必填”标记的字段，是当前操作最容易漏填、但运行时必须用到的内容。第一次使用时优先把这些字段填完整。

## 7. 常见结果卡片

- “任务向导”：由“流程状态”生成，显示下一步建议、每个步骤是否就绪、当前缺口数量。
- “项目总览”：汇总文献、笔记、图表、仿真导出和稿件数量，适合每次打开课题后先看。
- “体检概览”：把文献、仿真、稿件、项目资产分成健康卡片，并把 `Next Actions` 单独显示成“下一步建议”卡片；下方仍保留完整原始报告。
- “稿件检查概览”：汇总警告、提醒、引用问题、图表问题、结构问题、DOCX 图片替代文本缺口和修订/批注痕迹。
- “仿真数据概览”：显示数据行数、识别列数、校验状态、非数字列、样例行或数值范围。
- “图形生成结果”：列出生成的 `.svg` 和 `.json` 路径，并汇总图类型、尺寸、坐标列、配色、坐标范围、图例和网格设置。
- “路径助手”：由“扫描可用文件”生成，可以把扫描到的路径重新填到对应表单。
- “文献洞察”：汇总文献总数、已有摘要数量、缺失 PDF/笔记数量、高引用候选、关键词排行和数据库来源。
- “文献操作结果”：添加、导入、搜索、近期文献、按来源查找完成后显示当前条目数或命中数。
- “文献资产概览”：文献库统计、检查缺 PDF、检查缺笔记完成后显示缺失项和已找到资产。
- “报告已保存”：保存常用报告后显示报告类型、保存路径和用途。
- “规划报告概览”：生成写作素材包、写作看板、文献对比表、文献地图、追踪清单后显示报告区块、条目数量和建议用途。

## 8. 常用报告保存位置

如果你想把常用报告直接保存到课题目录，在“项目总览”里使用“保存常用报告”。默认保存位置是：

- 写作素材包：`manuscript/writing-pack.md`
- 写作看板：`manuscript/writing-dashboard.md`
- 文献对比表：`notes/literature-table.md`
- 文献地图：`notes/literature-map.md`
- 追踪清单：`notes/literature-tracker.md`

保存完成后，可以打开这些 `.md` 文件，复制到论文草稿或继续整理。
